# Security Policy

## Supported Versions

This project primarily supports the latest major release line (v9).

Security fixes are applied to the main branch and are included in subsequent releases.

Older versions may not receive security updates.

## Reporting a Vulnerability

If you believe you have found a security vulnerability in this project, please do not open a public GitHub issue.

Instead, report it privately using GitHub's Private Vulnerability Reporting feature.

If private reporting is not available, please contact the project maintainers through a non-public channel if one is available.

## What to Include

When reporting a vulnerability, please include as much of the following information as possible:

- a description of the issue
- affected versions, configurations, or usage patterns
- steps to reproduce the issue
- proof-of-concept code or requests, if available
- any suggested mitigations or fixes

## Disclosure Process

The maintainers will review the report and triage it.

If the issue is confirmed, a fix will be prepared and released as appropriate.

Public disclosure should be delayed until a fix or mitigation is available, when possible.
